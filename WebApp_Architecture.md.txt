# 2026 青年消费洞察看板：全量系统架构与交互协议 (V3.3 - With AI Search)

## 1. 界面板块架构 (Interface Layout)

系统采用“双层联动”布局，结合底部全局 AI 搜索悬浮组件，确保从宏观趋势到微观实体的无缝下探与深度问答。

### A. 宏观洞察层 (Macro Insight Hub)
* **定位**：页面上部（或左侧 40%），采用 Bento Grid 布局。
* **组件一：动机热力玫瑰图 (Rose Chart)**
    * **驱动**：`analysis_summary.json` -> `motivation_density`。
    * **动机字典约束**：必须严格映射至以下预定义的动机集合，不允许擅自通过分词切分产生计划外动机：
        * 智性决策：科技平权、信息平权、秩序与掌控感、生产力与效率、舒适感。
        * 情绪补偿：多巴胺、玄学叙事、精致美学、悦己主义、Gap、懒人经济、孤独焦虑、创造探索。
        * 身份叙事：Lifestyle、圈层共振、社交货币、小众DIY、文化根脉。
    * **数据映射与排序**：扇区面积大小和排列顺序必须严格依据 **核心GMV** 的真实数值计算，确保视觉比例与真实商业价值绝对挂钩。
    * **层级联动展示**：
        * 初始态（全景视角）：饼图仅展示【智性决策】、【情绪补偿】、【身份叙事】3大板块。
        * 选中某板块（如通过顶部 Filter）：饼图下钻展示该板块下属的动机。
    * **视觉与文本标签**：
        * **标题固定**：主标题“动机热力图”固定左上角；副标题（如“全景视角”或“身份叙事”）固定右上角，不带冗余后缀。
        * **静态标签 (Label)**：扇区外围直接且永久展示“名称 + 核心GMV + 百分比占比”。
    * **交互响应**：
        * **悬停 (Hover)**：显示 Tooltip，内容精简为板块名称/动机名称、**品牌数量**，以及 **Top 3 代表品牌信息**。
        * **点击 (Click)**：点击板块/动机扇区后，右侧 Entity Matrix 更新为该板块/动机下的品牌，并锁定状态。鼠标移出饼块时，右侧卡片状态不重置。
* **组件二：智性博弈散点图 (Bubble Plot)**
    * **标题与说明**：左上角主标题为“溢价散点图”；右上角动态展示副标题（“全景视角”或对应板块名）。
    * **动机字典约束**：与玫瑰图及柱状图一致，严格遵循全局动机字典，禁止自由分词。
    * **坐标与排序**：X轴-功能参数强度（打散逻辑下，气泡从左到右严格按照溢价率由低到高进行X轴坐标分配）；Y轴-溢价率 (Premium Rate)。
    * **层级联动展示**：
        * 全景视角：聚合展示三大板块的 3 个气泡，气泡大小代表该板块下的品牌数量，Y轴为该板块平均溢价率。
        * 钻取视角（选中某板块）：打散展示该板块下的所有字典内动机气泡。
    * **视觉与文本标签**：
        * **静态标签 (Label)**：静态展示“名称 + GMV + 溢价率”。
        * **悬停 (Hover)**：显示 Tooltip，展示名称、溢价率、品牌数量及 **Top 3 代表品牌信息**。
    * **功能**：点击气泡执行过滤联动（与玫瑰图点击逻辑一致）。
### 1.3 商业生态象限矩阵 (Ecological Quadrant Matrix)

该板块由“社交效能”排行榜与“生态象限”散点图组成，旨在通过社交声量与商业结果的剪刀差，推导出品牌的经营质量与竞争生态位。

#### A. 视觉映射与推理逻辑 (Data Reasoning)
1. **社交效能杠杆 (Conversion Leverage) - 动态能量柱模式**:
   * **逻辑**: $杠杆率 = \frac{GMV}{Social\_Mentions}$（系统模拟 Social Mentions = efficiency_score * 800）。
   * **表现**: 动态能量柱模式下，柱体长度代表杠杆百分比（如 120%），并按杠杆率降序排列。
   * **处方签诊断 (Diagnosis)**:
     - 杠杆率 >= 10.0 (1000%): **[强转化型]**，用户心智成熟，搜索即购买。
     - 3.0 <= 杠杆率 < 10.0 (300%-1000%): **[均衡型]**，声量与转化处于健康水平。
     - 杠杆率 < 3.0 (300%): **[苦力型]**，社交热度远超实际转化，属于品牌建设期。
     - *特例*：若声量 > 500 且溢价为负 $\rightarrow$ 标注 **[价格战受害者]**，代表该动机下品牌正在通过牺牲利润换取流量（优先级最高，带 🔥 标识）。

2. **双模式象限分析 (Dual-Mode ECharts)**:
   * **组件**: 提供一个切换开关（Toggle），点击后柱状图平滑散开转化为“生态象限散点图”。
   * **Mode A (社交效能)**: 
     - X轴隐去，Y轴为动机名称，柱体按 Leverage 排序。
     - 颜色映射：珊瑚色（强转化）、赤陶色（均衡）、冷灰色（苦力型）。
   * **Mode B (生态象限散点图)**: 
     - X轴 = 社交声量 (Social Volume)；Y轴 = 溢价率 (Premium Rate %)。
     - 气泡大小：映射该动机赛道下的 **品牌数量** (`Math.sqrt(brandCount) * 5`)。
     - 气泡代表的是**整个动机板块**，而不是单个品牌。
     - **十字坐标线 (MarkLine)**：在平均声量和平均溢价率处画十字准星，划分四大商业生态象限：
       - **[名利双收]**：高声量 + 高溢价 (右上)
       - **[闷声发财]**：低声量 + 高溢价 (左上)
       - **[内卷之王]**：高声量 + 低溢价 (右下)
       - **[长尾试错]**：低声量 + 低溢价 (左下)

---

#### B. 交互行为细节 (Interaction Detail)
* **动态切换 (The Morph)**: 当用户点击“生态象限”开关时，所有柱体执行平滑的“离散动效”（Duration: 700ms），演变为散点分布。
* **悬停透视 (Hover Insight)**: 
  - 柱状图模式：展示具体的 GMV、社交声量、溢价率，以及 Leverage 处方签。
  - 散点图模式：展示具体的 X/Y 轴数值、品牌数，以及对应的**四大生态位诊断结论**。

#### C. Agent 执行指令 (Action Directives)
1. **构建逻辑处理器**: 编写 `calculateLeverage()` 函数，对 `social_efficiency` 和 `motivation_density` (GMV) 进行二次计算，生成 Leverage 杠杆指标。
2. **实现视图转换器**: 利用 ECharts 的 Universal Transition（`universalTransition: true`）功能，实现“转化排行榜”到“溢价矩阵”的无缝 transition。
3. **视觉反馈定义**:
   - 柱体右侧使用 Rich Text 清晰排版 Leverage (加粗珊瑚色)、GMV (加粗黑色) 和 社交声量。
* **组件四：逻辑迁徙网络 (Force Graph)**
    * **交互**：双击连线显示跨动机桥梁品牌（如：观夏）。

### B. 微观实体层 (Entity Matrix)
* **定位**：页面主体区域，支持无限滚动。
* **组件一：无限滚动卡片墙 (Infinite Grid)**
    * **筛选器**：顶部常驻“智性决策 | 情绪补偿 | 身份叙事”板块切换及对应 Tag 筛选。
    * **卡片视觉**：品牌名、核心产品、板块色块背景。
    * **动效**：卡片悬停时向上位移 8px，边框呈“呼吸灯”光效。
* **组件二：品牌参数化面板 (Brand Modal)**
    * **唤起**：点击卡片后呈“液态扩散”展开。
    * **内容**：
        1. **六维雷达图**：标准化展示声量、GMV、溢价、情绪能量、逻辑强度、智性指数。
        2. **打字机处方**：逐字渲染 `prescription` 字段，带闪烁光标。
        3. **数字计数器**：GMV 数值从 0 快速翻滚增长至真实值。

### C. 智能交互层 (AI Search Component)
* **定位**：页面底部居中悬浮 (`fixed bottom-8`)，全局可用。
* **视觉元素**：
    * 极简高级感搜索框，遵循 Claude 设计系统的 Parchment 背景与 Warm Sand 悬停态。
    * **挂载助手 (Assistant Asset)**：搜索框右侧内置动效头像。
        * 静态 (Static)：`assistant_idle.png`
        * 动力态 (Active)：`assistant_thinking.gif`
* **交互逻辑 (Interaction Flow)**：
    * **输入与提交**：用户输入自然语言问题，按 Enter 提交。
    * **推理中 (Loading)**：头像切换为 `thinking.gif`，搜索框锁定防重复提交。
    * **流式输出 (Streaming)**：上方弹出气泡卡片，通过 Vercel AI SDK 接入 OpenRouter (Google Gemini 3.1 Pro Preview) 接口，使用 React Markdown 实时渲染流式回复。
    * **完成态 (Finished)**：回复结束，头像切回 `idle.png`，搜索框恢复可用。

## 3. 核心交互联动协议 (Interaction Protocol)

### 3.1 跨组件联动 (Cross-Component Sync)
* **触发点**：玫瑰图 Hover / 导航栏 Filter。
* **响应链**：
    1.  `filteredBrands` 全局状态更新。
    2.  `Entity Matrix` 执行卡片重排动画（Layout Animation）。
    3.  `Bubble Plot` 非选中气泡置灰，目标气泡闪烁。

### 3.2 细节反馈规范 (Micro-interactions)
* **动画曲线**：主要使用 `Cubic-bezier(0.34, 1.56, 0.64, 1)` 实现灵动的弹性效果。
* **响应时长**：
    * 卡片悬停反馈：即时 (< 50ms)。
    * 数据重排动效：400ms - 600ms。
    * 打字机速度：80ms/字。
